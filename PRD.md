# AutoFormat - 智能文档排版工具

> **产品名称**：AutoFormat（智能排版助手）
> **产品类型**：Web 单页应用（纯前端，无需后端）
> **核心定位**：将任意文档内容一键套用专业排版规范，告别逐段手动调格式
> **目标用户**：学生（论文/作业）、职场人（商业计划书/报告）、教师（教案/课件）

---

## 1. 市场与痛点分析

### 1.1 目标场景

| 场景 | 典型用户 | 痛点 |
|------|----------|------|
| 毕业论文 | 本科/硕博学生 | 高校格式规范繁多（字体/字号/标题层级/参考文献），每次手动调整耗时数小时 |
| 商业计划书 | 创业者/投资人 | 投资人偏好统一风格，商业计划书需专业排版，手动设置浪费精力 |
| 课程作业 | 中小学生/大学生 | 各科作业格式要求不同，重复设置浪费时间 |
| 政府公文 | 公务员/文职人员 | 公文格式严格（仿宋GB2312/黑体/固定行距），手动操作易出错 |
| 科研论文 | 科研人员 | APA/MLA/国标等不同引用格式，LaTeX门槛高，Word操作繁琐 |

### 1.2 现有解决方案的不足

- **WPS/Word 模板**：模板数量有限，需要手动套用，仍需逐段设置
- **在线排版网站**：功能单一，无法自定义复杂规则
- **LaTeX**：学习成本极高，普通用户难以驾驭
- **AI 写作工具**：重内容生成，轻排版，无法精准控制格式细节

### 1.3 我们的机会

一个**零学习成本**、**所见即所得**的排版工具——用户只需三步：
1. 粘贴或上传文档
2. 选择/配置排版规范
3. 一键导出

## 2. 产品功能设计

### 2.1 核心功能（MVP）

#### F1：文档输入
- **粘贴文本模式**：直接粘贴纯文本，工具自动识别段落和标题
- **上传 .docx 文件**：解析 Word 文档，保留段落结构
- **上传 .txt 文件**：作为纯文本处理

#### F2：排版规则配置
提供可视化的排版规则面板，支持以下维度：

| 配置项 | 可选值 | 说明 |
|--------|--------|------|
| **文档类型** | 论文 / 商业计划书 / 课程作业 / 政府公文 / 自定义 | 预设对应格式规范 |
| **中文字体** | 宋体 / 仿宋 / 黑体 / 楷体 / 微软雅黑 | 默认宋体 |
| **英文字体** | Times New Roman / Arial / Calibri | 默认 Times New Roman |
| **标题字号** | 一级~六级（对应小二~小五 / 14~9pt） | 可分别设置各级标题 |
| **正文字号** | 五号(10.5pt) / 小四(12pt) / 四号(14pt) | 默认五号 |
| **字体加粗** | 全部标题加粗 / 仅一级标题加粗 / 不加粗 | |
| **字间距** | 标准(0) / 稀疏(+0.5pt) / 紧凑(-0.25pt) / 自定义 | |
| **行间距** | 单倍行距 / 1.5倍行距 / 2倍行距 / 固定值(20pt/22pt) | |
| **段落间距** | 段前 / 段后（可设 0 / 0.5 / 1 / 1.5 行） | |
| **对齐方式** | 左对齐 / 居中 / 右对齐 / 两端对齐 | 默认两端对齐 |
| **首行缩进** | 无 / 2字符 / 4字符 | 默认2字符 |
| **页眉内容** | 文档标题 / 自定义文字 / 无 | |
| **页脚内容** | 页码 / "第X页共Y页" / 自定义 / 无 | |
| **编号样式** | 无编号 / 自动编号(1. 1.1 1.1.1) / 项目符号(• ● ✓) | |
| **装订线** | 左侧装订边距 2.5cm / 普通边距 | |

#### F3：智能标题识别
- 自动识别粘贴/上传文档中的标题（根据行末冒号、空行、前缀数字等特征）
- 用户可手动标注"这是一级标题"等
- 支持拖拽调整段落层级

#### F4：实时预览
- 左侧编辑区 → 右侧预览区，实时渲染排版效果
- 所见即所得，无需反复导出查看

#### F5：一键导出
- 导出为标准 .docx 文件（可在 WPS/Word 中直接打开）
- 导出为 .pdf（基于 docx 转换）
- 导出为 .txt（纯文本，排版信息丢失）

#### F6：预设模板库
预置常用排版规范，一键应用：

| 预设名称 | 适用场景 |
|----------|----------|
| 📚 高校学位论文（GB/T 7714） | 本科/硕士/博士论文 |
| 📋 商业计划书（标准） | 投资人路演 |
| 📝 课程作业（通用） | 中小学/大学作业 |
| 📄 政府公文（仿宋） | 机关公文 |
| 📰 科研论文（APA 7th） | SCI/SSCI 期刊 |

### 2.2 迭代功能（V2+）

- **F7：格式刷** - 选中一段文本，将其格式复制到其他段落
- **F8：批量替换格式** - 将全文所有"宋体12pt"替换为"楷体11pt"
- **F9：云端模板市场** - 用户上传/下载他人分享的模板
- **F10：Word插件** - 集成到 WPS/Word 作为插件使用

## 4. 技术方案

### 4.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | Vite + React 18 | 快速构建，单页应用 |
| 样式 | Tailwind CSS | 原子化 CSS，快速布局 |
| 文档解析 | mammoth.js | 将 .docx 解析为 HTML |
| 文档生成 | docx.js | 按规则生成标准 .docx |
| 文件导出 | file-saver | 触发浏览器下载 |
| 图标 | lucide-react | 轻量图标库 |

### 4.2 系统架构（单文件 SPA）

```
App
├── Header（标题栏 + 导出按钮）
├── Main（三栏布局）
│   ├── ConfigPanel（左侧：排版配置）
│   │   ├── TypeSelector（文档类型预设）
│   │   ├── FontSettings（字体字号）
│   │   ├── ParagraphSettings（段落间距）
│   │   ├── HeaderFooter（页眉页脚）
│   │   └── NumberingSettings（编号样式）
│   ├── Editor（中间：文档输入）
│   │   ├── UploadZone（拖拽上传）
│   │   └── ContentEditor（内容编辑）
│   └── Preview（右侧：实时预览）
└── Footer（状态栏）
```

### 4.3 关键技术点

**文档解析流程**：
```
.docx 文件 → mammoth.js → HTML DOM → React State → 渲染预览
```

**排版生成流程**：
```
用户配置 → 生成 docx 规则对象 → docx.js 生成 .docx → file-saver 下载
```

**标题智能识别算法**：
- 检测行首模式：`^第[一二三四五六七八九十]+[章节条]`, `^\d+[\.、]`, `^[一二三四五六七八九十]+、`
- 检测空行前后关系：空行分隔的短行 → 可能是标题
- 结合字号差异（如原文档字体更大）→ 推断为标题
